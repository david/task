# Local rules for `/skill:task`

Use this repo's `task` CLI through `bun task.ts` from the repo root.

## Local rules

- Follow `AGENTS.md` and the docs in `doc/` as the authoritative local contract.
- Prefer the canonical issue document paths and workflow notes in `doc/task-workflow.md` when issue-backed skills coordinate through `research/`, `tasks/`, `task-status/`, `code-history/`, `check-report/`, `taskify-history/`, `qa-results/`, or `qa-context/`.
- Prefer supported commands documented in `doc/commands.md`; do not invent undocumented commands.
